# Say "conform", never "sweep"

For bringing code or prose into line with a convention, say you **conform** it to the convention — never "sweep". (Where they read more exactly, *reconcile* or *propagate* also work.)

**Why:** "sweep" is a loose figure that says nothing about the intent; "conform" states the actual thing — bringing something into line with a convention. Literal naming over slang (a case of the no-slang standard).

**How to apply:** write "conform" (or "reconcile" / "propagate" where more exact) wherever the impulse is "sweep", in names, prose, comments, and commit messages. Related: the no-slang rule and the name-literally-not-by-analogy rule.
