# Say "obsolete", never "dead"

Never use **"dead"** to convey that something is no longer used, inactive, or obsolete — no "dead path", "dead reference", "dead link", "dead branch", "dead code". Say **obsolete**, unless a more exact word is available.

**Why:** "dead" is slang, and it over-dramatizes — it imports a finality and a drama the situation does not have. Nothing died: a reference whose target moved has simply stopped applying, and that is what **obsolete** states literally. The exaggeration also distorts judgment, making a routine tidying read as urgent repair. Literal naming over a figure of speech (a case of the no-slang standard).

**How to apply:** write "obsolete" wherever the impulse is "dead". Reach for the more exact word when there is one — the point is to say what is actually true of the thing, and "obsolete" is the fallback, not the goal:

- a reference whose target moved → **obsolete**, or **unresolvable**
- a rule no longer in force → **retired**, or **superseded**
- a branch that can only go one way → say that, or that it **has nothing to decide**
- an assignment that never happens → it **never fires**
