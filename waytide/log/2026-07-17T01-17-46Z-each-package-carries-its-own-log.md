# Established that each package carries its own decision log (`<package>/log/`), traveling with the package when a project installs it. A decision about a package's own rules is provenance that belongs with the package; because the packages are open source, distributing the reasoning alongside the rules is a feature, not a leak, and the nesting under `agent/rules/<package>/log/` namespaces it cleanly from a consuming project's own `agent/log/`. This corrects an earlier over-cautious stance that authoring metadata had to be kept out of distributed content. Two log concepts are kept distinct: a **project's** own decision log (the foundation convention — `agent/log/` for the project's decisions) and a **package's** authoring log (`<package>/log/`, its provenance). The `subject-first` re-home entry was moved into `git/log/` and the git package re-split and pushed as the first instance. Composite-level framework entries (`include:`/`package.md`, the ISO-prefix exemption, the composite/component naming, and this one) relocate into `foundation/log/` when the foundation package is built; the composite top-level `agent/log/` then keeps only transitional migration-milestone records. Folded into the foundation migration plan.
