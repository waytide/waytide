# Rebranded the project from "Agent Norms" to **Waytide** (domain `waytide.ai`, registered by Scott). The framework is general-purpose, not Eventide-specific, so it gets its own identity: distribution moves from the `eventide-project` org to a new **`waytide`** org — the composite becomes `waytide/waytide`, and each component repo becomes `waytide/<package>` (dropping the `agent-norms-` prefix; `code/ruby` → `waytide/code-ruby`). "Waytide" won over Precepts / Canon / Craft / normcraft / flowcanon / tidecanon etc.; chosen partly because `.ai` single words are all taken and `waytide.ai` was available, and "way" fits — the framework is a *way of working* the agent follows, with "tide" nodding to the Eventide lineage. Did the composite-side text rebrand now (Phase A): the composite and package READMEs, `install-all.sh`, `foundation/install.sh`, the three `install-dependencies.sh`, CONTRIBUTING, the packaging design, the state-of-the-work note, and the website drafts — all "Agent Norms" → "Waytide" and all repo URLs → `github.com/waytide/<package>`. The `eventide-project/constant` source references stay (Constant genuinely lives there). Historical `agent/log/` entries and the completed migration plans keep the old name; the phase-b runbook and checklist got a warning banner (their old-org commands must not be run as-is). The first Waytide release — creating the `waytide` org and repos and publishing fresh (no fast-forward; new org, new names) — is a Phase B step folded into the pending-release deferred item; `github.com/waytide` and the domain are confirmed available/registered.
