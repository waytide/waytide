# **Added the `say-obsolete-not-dead` rule to `vocabulary`** — never use "dead" to convey that something is no longer used, inactive, or obsolete (no "dead path", "dead reference", "dead link", "dead branch", "dead code"). It is slang and it over-dramatizes: nothing died, a reference whose target moved has simply stopped applying, which is what **obsolete** states literally; the exaggeration also distorts judgment, making a routine tidying read as urgent repair. **"Obsolete" is the fallback, not the goal** — the rule directs you to the more exact word where one exists (a reference is *unresolvable*, a rule *retired*, a branch *has nothing to decide*, an assignment *never fires*). Tenth one-substitution rule under the `no-slang` principle; the README's substitution table gains the row. **Consequences:** conformed this session's composite `agent/log/` entries; found the published `say-verified-not-green` closing with "existing records are not retroactively **swept**", violating `vocabulary`'s own `say-conform-not-sweep` (now "conformed"); and conformed two `code/ruby` rules — `build-constructor-normalizes` ("only adds a dead branch" → "only adds a decision with one possible outcome") and `optional-params-default-in-body` ("makes the receiver's own `||=` dead" → "the receiver's own `||=` never fires"). **`code/ruby` now requires a re-publish too** — five of the six published packages do.
