# Renamed the error-test naming: an `assert_raises` test is named **"Fails"** (was "Is an error"), and its non-raising complement **"Doesn't fail"** (was "Is not an error"); the condition stays promoted to a `context`. "Fails" / "Doesn't fail" name the actuation's behavior directly. Because "Fails" does not start with "Is", it no longer clashes with the "Is-only-for-value-equals" rule — which was the sole reason the naming had been folded into `test-name-is-prefix` (M4). So it **unfolds** back into its own rule (`…-error-test-named-fails-condition-is-context.md`), fully consistent with (not an exception to) the "Is"-naming rule; `test-name-is-prefix` returns to just the "Is" rule with the error case as a plain non-"Is" outcome.
