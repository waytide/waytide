# **Pre-Phase-B conformance sweep — every package's distributed prose checked against the substitution and de-slang rules in its include graph.** **Headline: the "a package contradicts a rule it includes" fault class is now clean** — no package violates a rule its own graph carries. This session's earlier fixes (green-on-arrival exception, arrange, swept, dead, the codes) had closed the instances; the sweep confirms none remain, including DBE (the newest, deepest include graph) against `testing`'s and `language`'s rules. Two findings survived triage, both outside the strict fault class, resolved with Scott:

- **`git` used the project's rejected slang, bound by no rule.** "green" for *tests pass* (`run-suite-before-commit` ×2, README ×1), "red" for *failing* (×2), "guard" for the *suite's protective role* (×1). `git` is standalone (`include: (none)`), so `testing`'s verified-not-green / protect-not-guard don't formally bind it — but it is the project's own discipline. **Decision: conform it anyway** — green→passes/verified, red→failing, guard→protection. The exempt compound **green-on-arrival** (in a commit-message example) stays.
- **`testing` said "wrap a test in a context."** `say-mediates-not-wrap` is in `testing`'s graph but scoped to mediating/accessor *objects*, which a context is not — so its literal reach didn't cover this. **Decision: change the verb to "enclose"** (and "holds" for the containment sense). "Enclose" names the act — one context around one test — while **"nesting" is kept for hierarchical depth** (nested context, nesting level, over-nested), the sense the package already uses ("context nesting mirrors folders"). Using "nest" for the single-enclosure act would have overloaded that. Conformed across `context-only-for-local-instrumentation` (title + body), its README summary line, and the error-test rule.

**Not touched (correctly):** DBE's red-green-refactor / red/green-loop / green-on-arrival (the cycle-and-ceremony exemption to verified-not-green); "guard clause" and code-guards in `testing` (the rule exempts the code construct); "arrange" used descriptively (Scott's ruling); "factory" for the constructor pattern in `code/ruby` (the controls-not-factories rule is scoped to test controls, and `code/ruby` doesn't include `testing`). **The standing rule reaffirmed:** a package need only conform to rules its own include graph carries; `git`'s conformance is voluntary adherence to the project standard, not an induced dependency.
