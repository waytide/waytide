# Published the first two Agent Norms packages and proved the subtree distribution workflow end-to-end. Created public repos `eventide-project/agent-norms-docs` and `eventide-project/agent-norms-git`, produced a one-package view of each with `git subtree split --prefix=<package>`, and pushed each split to its repo's `main` — the one-directional composite repo → component repo sync the design settled on. Verified the consuming side by `git subtree add --prefix agent/rules/git …` into a throwaway project: all six files (the four rules, `README.md`, `package.md`) landed at `agent/rules/git/` with names intact, so the session-start "read every file in `agent/rules/`" behavior is preserved. The temporary `split-docs`/`split-git` branches are throwaway (a release re-splits from the composite repo) and were deleted. This is Phase B of the docs+git migration plan and closes the design's "prove the subtree workflow" next-step. Push auth: `gh` wired in as git's credential helper (`gh auth git-credential`).
