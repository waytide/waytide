# Each package that has dependencies carries an `install-dependencies.sh` at its root (`language`, `testing`, `code/ruby`, `design-by-efferent`) — a POSIX sh script that `git subtree add`s each included package into the consuming project's `agent/rules/`, idempotently (skips a dep already present). Run from the project root. `foundation`, `git`, `docs` are standalone and carry no script. Amended foundation's file-name prefix rule to exempt `install-dependencies.sh` as a fourth structural file (alongside `package.md`, `README.md`, `vocabulary.md`) — named for what it is, not a dated artifact. Each dependent package's README references the script. Re-published foundation (rule) + the four script-carrying packages.
