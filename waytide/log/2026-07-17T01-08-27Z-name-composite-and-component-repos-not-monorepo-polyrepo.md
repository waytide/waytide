# Named the two repository roles **composite repo** (the one holding every package, where authoring happens) and **component repo** (a repo holding one package), and retired the terms "monorepo" and "polyrepo" from the project. A component repo is just a repo; the mono-/poly- vocabulary is imported jargon that obscures a plain composition relationship — a whole built from components — and it fails the project's own name-literally / no-slang discipline. "Composite" and "component" name the structural relationship directly. Conformed all live docs (README, packaging design, handoff, migration plan) and the prior decision-log entries; the one remaining "distribution-repos" token is a filename slug, left as-is so the record's link stays intact. Applies to prose and conversation, not only the artifacts.
