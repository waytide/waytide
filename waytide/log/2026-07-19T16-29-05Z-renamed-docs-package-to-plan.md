# Renamed the `docs` package to `plan` (`docs/` → `plan/`). "docs" was too broad — it reads as general documentation (API docs, user guides, READMEs), but the package only governs how a project's plan and design documents are written (its four rules: plan-document-format, design-document-format, plans-no-code-samples, package-dependency-heading). Considered `planning`, `plans-and-designs`, and `authoring`; Scott chose `plan`. Despite the singular name, it covers design documents too — the package README and the composite README say so outright. Avoided naming it `design` because that would collide with the `design-by-efferent` package. Conformed all live docs: the composite README (package list, dependency graph, install blocks), `install-all.sh`, the package README, `foundation`'s agent-plans-convention / agent-design-convention / README cross-references, the packaging design, the classification table, the state-of-the-work note, and two deferred items. Historical `agent/log/` entries, the completed migration plans, and the 2026-07-17 Phase B runbook record keep the old name — they record what was true then; this entry is the link between the names. The component repo is still `agent-norms-docs`; renaming it to `agent-norms-plan` (with a history reset, since the composite path changed `docs/`→`plan/`) is a Phase B step folded into the pending release, exactly like the earlier `vocabulary` → `language` rename.
