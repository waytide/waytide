# The three phases of a test are **control → actuate → observe**, not control → actuate → assert. The third phase is *observing* the outcome; an assertion is only the mechanism by which the outcome is observed (a test can also observe via narration — comment/detail). This corrects the "control → actuate → assert" framing carried in the source `say-control-not-arrange` rule. The `assert()` call itself keeps its name — this is about the phase, not the mechanism.
