# **Renamed the `vocabulary` package to `language`.** A vocabulary is a set of words, and the package was never that: `no-slang` governs phrasing in prose and conversation, `name-literally` governs how a name is *constructed*, and even the substitutions are stance-installing rather than lexical. The surface these rules act on is language use — which is what `foundation`'s `rules-install-a-mindset` prescribes filing by. **The collision was the decisive argument:** DBE's `vocabulary.md` *is* a glossary — terms with meanings, the thing the word properly names — so two different things were called vocabulary, one of them the subject of a prefix-exemption rule written hours earlier. `language` frees the word: any package may carry a `vocabulary.md`; the `language` package governs how language is used everywhere, and its README now says so outright. **The tell was a homeless statement** — "use of language is very precise under this model" had nowhere to sit in a package named for one part of its own subject; it is now the package's governing principle (`language-is-precise-here`), with `name-literally` and `no-slang` beneath it as the two disciplines that make precision achievable. **Conformed:** three `include:` lines (`testing`, `code/ruby`, `design-by-efferent`), cross-references across five packages, the composite README's package list and dependency graph, the packaging design, the classification table, and the handoff. `local/vocabulary/` → `local/language/`. The component repo becomes `agent-norms-language`. **Also disambiguated `code/`**, described as "a language grouping" — it groups by *programming* language, and now reads that way. Historical `agent/log/` entries keep the old name: they record what was true when written, and this entry is the link between the names. **Timing:** done now because five packages already needed re-publishing, so the rename cost only the edits; once anything installs the package it would have been a migration.
