# Adopted `include:` (not "Depends on") as the package-dependency declaration, carried in a per-package `package.md` file kept separate from the informative `README.md` (Agent Norms composite repo). "include" fits the Ruby mixin idiom the project already uses (a package includes its dependencies the way a class mixes in a module) and the subtree mechanic (including a package pulls in its dependency subtrees). Separating `package.md` (the mechanical declaration) from `README.md` (a human-facing introduction) keeps each file to one job. Both structural files are exempt from the ISO-8601-UTC filename convention. Filename `package.md` is provisional — kept for now, open to a better term later. Applied across the design docs and realized in the docs/git packages during Phase A.
