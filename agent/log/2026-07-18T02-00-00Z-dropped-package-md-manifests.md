# Dropped the per-package `package.md` manifests. Their sole job was the one-line `include:` dependency declaration, which the `install-dependencies.sh` scripts (added 2026-07-18) now carry in executable form — a package with dependencies has the script; a standalone one has neither. Deleted all seven `package.md` files. Amended foundation's file-name prefix rule (removed `package.md` from the structural-file exemption; the exempt set is now `README.md`, `vocabulary.md`, `install-dependencies.sh`). Updated the README (dependency-tree intro and the install note), the packaging design's "no manifest" decision, the handoff, and the Phase B checklist/runbook prove-the-pull checks. Tradeoff accepted: dependency info is now imperative (in the script) rather than a declarative manifest, and "no dependencies" is signaled by the script's absence. Historical migration plans keep their `package.md` references. Re-published all seven packages.
