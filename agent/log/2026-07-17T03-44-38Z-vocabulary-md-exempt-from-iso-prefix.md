# **Amended foundation's file-name prefix rule to exempt `vocabulary.md`.** The exemption previously named `package.md` and `README.md` by list; `vocabulary.md` (a package's glossary, where it has a cohesive lexicon) is a third structural file the list did not cover, so DBE's glossary would have been a naming violation on arrival. Restated the exemption by its principle rather than as a longer list — a package's **structural** files are named by what they are (a manifest, an intro, a glossary), while its **dated artifacts** (rules, `log/` entries) always take the prefix. **`foundation` now requires a re-publish** (Phase B, gated). Also removed the constant-specific worked-promotion path from `name-literally` and trimmed obsolete `agent/rules/**` cross-references throughout the DBE rules, per the established convention: rules reference each other by name, not by path.
