# Moved the `communicate-plainly-name-the-thing` rule from `foundation` to `language`. It was first put in `foundation` to be always-on (foundation is installed everywhere, `language` is optional), but a general "how to talk" rule is out of place in the framework-mechanics package. It is a rule about how language is used — precisely the `language` package's subject — so it now sits there as a third principle rule beneath the `language-is-precise-here` premise, alongside `name-literally` and `no-slang`. Reframed accordingly (dropped the "foundation is always present" rationale; tied its reach-into-conversation to `no-slang`); the language README's principle-rule list gains it ("two disciplines" → "three"). Tradeoff accepted: it is no longer guaranteed present in a project that installs only `foundation` — thematic fit won over always-on.
